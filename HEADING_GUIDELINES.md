## Best Practices

### 1. Proper Hierarchy
```
# H1 (reserved for blog title, don't use manually)

## H2 - Main sections (4-6 per post)
Content paragraph explaining the section...

### H3 - Subsections (2-4 per H2)
More detailed explanation...

#### H4 - Sub-subsections (max 2 per H3)
Very specific details...
```

### 2. Structure Rules

**✅ DO:**
- Use H2 for major topics
- Add H3 under every H2 that has multiple paragraphs (3+)
- Keep H3s to 2-4 per H2 section
- Use H4 only when content truly needs a 4th level
- Use consistent language patterns:
  - H2: Questions or broad topics ("¿Qué es...?", "Cómo...", "Principales...")
  - H3: Specific aspects ("Características", "Ventajas", "Casos de uso")

**❌ DON'T:**
- Skip hierarchy levels (jump from H2 to H4)
- Use single words as headings
- Create H3 for single-paragraph sections (fold into H2)
- Use H4 without an H3
- Use more than 2 H4s in a section

### 3. Readability Guidelines

**Optimal section length:**
- H2 section: 300-500 words max before adding H3
- H3 section: 150-200 words max before adding H4
- H4 section: 100-150 words (minimal)

**Signs you need more headings:**
- Section is longer than 4 paragraphs
- Section covers 3+ distinct topics
- Section has multiple bullet lists (break with H3)
- Users can't scan content in <30 seconds

## SEO Impact of Proper Headings

### Rankings
- ✅ **Better featured snippets** — Clear H2/H3 structure matches Google's preferred patterns
- ✅ **Higher CTR** — Better formatted pages get more clicks from search results
- ✅ **Longer session time** — Users can scan and find content faster

### Crawlability
- ✅ **Clear structure** — Search engines understand content hierarchy
- ✅ **Better indexing** — Subheadings create internal anchor points for indexing
- ✅ **Topic relevance** — Heading pattern signals main topics to Google

### User Experience
- ✅ **Scannable** — Users grasp structure in <10 seconds
- ✅ **Accessible** — Screen readers use headings for navigation
- ✅ **Mobile-friendly** — Better outline on small screens

## Implementation Checklist

When creating or editing posts:

- [ ] Does post have 4+ H2s? Yes → Plan for H3s
- [ ] Does each H2 have 3+ paragraphs? Yes → Add H3
- [ ] Do H3s cover distinct subtopics? Yes → Keep them
- [ ] Is any H4 without an H3 parent? No → Good!
- [ ] Can someone scan in 30 seconds? Yes → Structure is good
- [ ] Do headings answer "what", "how", "why"? Yes → Clear messaging

## Template for Adding H3s

**Find posts with:**
- More than 5 H2s without H3 subsections
- Long text blocks (5+ paragraphs) under single H2

**Add H3 by:**
1. Identifying natural topic groups within H2
2. Creating H3 that name the subtopic
3. Moving content under appropriate H3
4. Ensuring H3s are parallel (similar grammatical structure)

**Example:**
```
# BEFORE
## How to Protect
[10-paragraph explanation covering 3 different topics]

# AFTER
## How to Protect
### Technical Controls
[3-4 paragraphs on firewalls, patches, etc.]

### Process Improvements
[3-4 paragraphs on policies, procedures, etc.]

### Employee Training
[2-3 paragraphs on awareness, drills, etc.]
```
